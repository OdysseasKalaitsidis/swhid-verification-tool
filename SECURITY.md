# Security Policy

## Supported Versions

Only the latest release of the SWHID Verification Tool is actively supported for security updates.

| Version | Supported |
| ------- | --------- |
| < 0.1.x | No        |
| 0.1.x   | Yes       |

## Reporting a Vulnerability

We take the security of this project seriously. If you find a security vulnerability, please do **not** report it via a public GitHub issue. Instead, follow the responsible disclosure process below:

1. Send an email to the project maintainers describing the vulnerability.
2. Include a detailed description of the vulnerability, a Proof of Concept (PoC) if available, and any details needed to reproduce or understand the risk.
3. We will acknowledge receipt of your vulnerability report within 48 hours and work with you to analyze and resolve the issue.
4. Once a fix has been prepared and tested, we will publish a new release and coordinate the disclosure.

Please give us reasonable time to investigate and fix the issue before publicizing it.
